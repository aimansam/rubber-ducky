# Testing Notes

## Lab Preparation

- Confirm the target is owned or explicitly authorized.
- Snapshot or back up the target before testing.
- Close sensitive applications and remove secrets from the desktop/session.
- Start endpoint logging or SIEM collection before inserting the device.
- Record OS version, username type, endpoint controls, and device details.

## Baseline Test

Objective: verify that a USB HID device can type a harmless message into a controlled text editor session.

Expected behavior:

- A text editor opens manually by the operator.
- The HID device types a short lab message.
- No commands are executed.
- No files are created beyond an optional local note saved by the operator.
- Logs capture device insertion or keyboard activity where available.

## Evidence to Capture

- Date/time of insertion.
- Device name, serial, or hardware identifier if available.
- Endpoint logs for new HID device connection.
- Screenshot of harmless typed message.
- Any alert generated by endpoint or SIEM tools.

## Blue-Team Validation Ideas

- Alert when a new USB keyboard appears on sensitive workstations.
- Alert when a keyboard device types with unrealistic speed after insertion.
- Review endpoint device-control policy coverage.
- Test user reporting process for unknown USB devices.
- Compare logs across Windows, Linux, and macOS lab hosts.
