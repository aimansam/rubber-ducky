Purpose: keep the blue-team detection work for the Pi Pico/Rubber Ducky lab inside the same Rubber Ducky repository.

This area focuses on defensive visibility, detection, response, and sanitized alerting for USB HID activity generated during authorized local lab testing.

## Scope

- Owned lab hosts and VMs only.
- Sanitized logs and evidence only.
- Discord is allowed only for defensive alert notifications.
- No C2, implants, target data collection, credential collection, file transfer, persistence, exfiltration, or remote control.
